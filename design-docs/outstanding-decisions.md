# Outstanding Decisions

This document holds decisions that are important enough to preserve but not settled enough to encode as doctrine. Service specs and build plans should reference these as open decisions instead of quietly choosing a local rule.

No top-level cross-cutting decisions are currently recorded here.

Resolved architecture work is indexed in `architectural-decision-log.md`.
Workstream and boundary specs should record open questions only when they
still block implementation. This file is a short index only; use the owning doc
for detailed unresolved items and close them there.
